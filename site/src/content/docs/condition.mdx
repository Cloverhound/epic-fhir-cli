title: Condition
description: Query problems, diagnoses, and health concerns.
---

The `condition` command manages Condition resources — problems, diagnoses, and health concerns.

## Commands

### `condition get <id>`

Fetch a single condition by ID.

```bash
fhir-cli condition get abc123
```

### `condition search`

Search for conditions.

```bash
fhir-cli condition search --patient e63wRTbPfr1p8UW81d8Seiw3
fhir-cli condition search --patient e63wRTbPfr1p8UW81d8Seiw3 --clinical-status active
fhir-cli condition search --patient e63wRTbPfr1p8UW81d8Seiw3 --category problem-list-item
```

| Flag | Description |
|------|-------------|
| `--patient` | Patient FHIR ID |
| `--clinical-status` | Clinical status: `active`, `inactive`, `resolved` |
| `--category` | Category: `problem-list-item`, `encounter-diagnosis` |
| `--code` | Condition code (ICD-10, SNOMED) |
| `--onset-date` | Onset date range |
| `--count` | Maximum number of results (default: `20`) |

### `condition active <patient-id>`

Shortcut to get active conditions (pre-sets `clinical-status=active`).

```bash
fhir-cli condition active e63wRTbPfr1p8UW81d8Seiw3
```

| Flag | Description |
|------|-------------|
| `--count` | Maximum number of results (default: `20`) |

### `condition problems <patient-id>`

Shortcut to get the problem list (pre-sets `category=problem-list-item`).

```bash
fhir-cli condition problems e63wRTbPfr1p8UW81d8Seiw3
```

| Flag | Description |
|------|-------------|
| `--count` | Maximum number of results (default: `20`) |

## Table Output

When using `-o table`, condition bundles display:

| Column | Description |
|--------|-------------|
| ID | FHIR resource ID |
| Condition | Condition code display text |
| Clinical Status | Active, inactive, or resolved |
| Onset | Onset date |
| Recorded | Date the condition was recorded |
